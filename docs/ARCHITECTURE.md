# AutoVideo 架构与方法论

> 概念级文档。说明系统怎么分层、为什么这么分。具体实现参数 / 配方 / 文案模型见完整版（[autovideo.fengyunagent.xyz](https://autovideo.fengyunagent.xyz)）。

## 一句话

不是「生成一条视频」，而是「**生成一条能批量产某类视频的产线**」。被生产的不是视频，是**产线本身**。

## 三层 + 两道闸门

```
┌──────────────────────────────────────────────────────────────┐
│  A 测量层（只提取，不评判）                                      │
│    数字层：镜头时长 / 卡点命中 / 字幕坐标 / 配色 / 语速 / 转场    │
│    文案层：转录 + OCR → 清洗 → 切分（钩子 / 正文 / CTA）→ 归属   │
│    原则：测量层只回答「说了什么、长什么样」，不回答「为什么爆」    │
├──────────────────────────────────────────────────────────────┤
│  B 推演层（核心 · CC 亲自下场的 harness loop）                   │
│    读测量 + 缩略图 → 假设剪辑配方 → 不确定就调研/用工具          │
│    → 比对「供料能力清单」（可行性闸门）→ 照配方出样片            │
│    → 和对标模板并排比 → 改配方 ↺                                │
│    收敛判据：样片 ≈ 模板（能冒充同款）                          │
├──────────────────────────────────────────────────────────────┤
│  D 脚手架层                                                     │
│    照锁定的配方，生成一个能跑的 <赛道>-pipeline 项目             │
│    内含生产 harness：writer → critic（继承现有生产管线）        │
└──────────────────────────────────────────────────────────────┘

闸门：关1 机械数字对照（lint，客观）+ 关2 真人「像不像 / 有没有魂」裁决
      用在线表格（多维表格）状态字段驱动：待审 / 通过 / 打回
```

## 为什么这么分

### 1. 内容是最高权重，不是工艺平级项
content-driven 视频（有口播 / 台词 / 字幕）里，**选题 + 文案是决定爆不爆的魂**。工艺（卡点 / 转场 / 配色）好测，所以容易显得重要；内容难测，但最重要。

性质差异：
- 工艺逆向出**常量**（这个时长、这种转场）。
- 内容逆向出**生成能力**（选题策略 + 文案 writer + 文案 critic），不是某一条具体文案。

### 2. 测量与分析必须分开
测量层只「提取事实」——客观、不带质量判断，避免污染。  
「为什么这么剪能爆」的分析推演，全部放在 B 层。  
混在一起，测量就会被结论带偏。

### 3. 收敛判据是「能复现」，不是「报告好看」
逆向对不对，唯一硬标准：**照推演出的配方重新生产，能不能复现模板**。  
样片冒充不了同款 → 配方就是错的，回去改。

### 4. 供料约束 = 诚实边界
三种料稀缺度天差地别：
- **转场 / 动效**：几乎不缺（代码无限供）。
- **BGM**：不缺（按 BPM + 情绪功能替代，本就不能用原曲）。
- **素材**：唯一真瓶颈。

> 可复刻边界 = 素材能否生成 / 抓取。  
> 截图流 / AI 图文 / 库存 B-roll / 卡点截图 → ✅ 可做（无脸赛道）。  
> 实拍出镜 / 探店 / 明星综艺 → ❌ 做不了。

**可行性闸门卡在 B 层**：推演阶段就告诉你够不够米，不到生产才发现是空厂。

### 5. 两道闸门，机器算数字、真人判味道
- **关1（lint，客观）**：样片和模板跑同一套测量工具，逐维比（镜头时长 / 卡点命中 / 字幕坐标 / 配色 / BPM…），落在模板自身实测区间内才算过。防 p-hacking。
- **关2（真人裁决）**：盲测——样片混进模板里，分不分得出？像不像出自这个号？说不清的「味道」问题，靠真人点出来，再转成新的可测维度。

## 数据先行（反臆测）

任何「列 N 种模式 / 结构 / 风格」的归纳，**先跨 ≥2 条同号视频验证一致**，否则只标「待验证」，不写进配方。  
缺信息默认去调研（全网 + 同号更多样本），不凭空设计。

## 两层 harness 分清

- **Loop 1（推演层）**：CC 亲自跑 + 调研 + 用工具，收敛 = 样片 ≈ 模板。
- **Loop 2（生产层）**：造好的厂自己跑 writer → critic，批量出片。
